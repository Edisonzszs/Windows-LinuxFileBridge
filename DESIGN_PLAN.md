# Windows-Linux 文件桥设计规划

## 1. 当前定位
这是一个面向个人高频办公场景的 Windows 本地文件桥工具，目标不是“功能尽可能多”，而是“Windows 和 WSL Linux 之间的常见文件操作足够顺手、足够安全”。

## 2. 当前版本的问题
- 能完成传输，但操作路径偏长
- 关键动作分散，传输、删除、定位没有形成连续操作流
- 缺少状态感，不容易一眼看出“我现在在哪、要传到哪、上次做了什么”
- 启动体验不够像桌面程序，带命令行窗口

## 3. 设计目标
- 少操作：高频动作尽量 1 到 2 次点击完成
- 少判断：界面尽量让用户不用想“这边是源还是目标”
- 更安全：删除、覆盖、批量操作必须有明确确认
- 更像桌面工具：启动干净、布局稳定、状态明确

## 4. 建议的信息架构
### 4.1 主界面分区
- 顶部状态栏
  - 当前 WSL 发行版
  - 当前模式：手动传输
  - 最近一次操作结果
- 中间双栏文件区
  - 左侧 Windows
  - 右侧 Linux
  - 每侧都包含：当前路径、返回上级、刷新、类型筛选、文件列表
- 中间操作条
  - 发送到右侧
  - 发送到左侧
  - 新建文件夹
  - 删除
- 底部任务与历史区
  - 最近操作记录
  - 失败重试
  - 错误详情

### 4.2 推荐交互原则
- 双击文件夹进入
- 单击选中
- 右键执行管理动作
- 主按钮只做传输，不做混合操作
- 危险动作统一弹窗确认

## 5. 功能分层建议
### 第一层：高频核心功能
- 双向复制
- 拖拽传输
- 删除
- 重命名
- 打开所在位置
- 查看属性
- 文件类型筛选

### 第二层：效率增强功能
- 新建文件夹
- 收藏常用目录
- 最近路径历史
- 搜索文件名
- 按修改时间/大小排序
- 批量选择反选

### 第三层：专业能力
- 覆盖策略设置
- 传输前后校验
- 同步预览
- 定时同步
- 冲突解决中心
- 日志导出

## 6. 我建议的下一轮迭代顺序
1. 新建文件夹 + 搜索框 + 排序
2. 收藏目录 + 最近目录
3. 真正可用的拖拽体验
4. 传输进度条和状态栏
5. 同步模式设计

## 7. 界面优化建议
- 顶部增加醒目的“当前源目录 / 当前目标目录”状态提示
- 中间两个传输按钮改成更短更明确的文案：`发送到 Linux`、`发送到 Windows`
- 文件列表增加列式信息，至少显示名称、类型、修改时间
- 历史区改成可折叠，默认只显示最近 5 条
- 危险操作按钮使用更明显颜色区分

## 8. 技术演进建议
### 继续用 Tkinter 的优点
- 已有代码可继续复用
- 适合个人桌面工具
- 依赖少，启动快

### 如果追求更现代体验
后续可以迁移到以下方案之一：
- PySide6：更像正式桌面软件，控件更丰富
- Tauri + Web 前端：界面最灵活，但开发复杂度更高

当前阶段我建议继续使用 Tkinter，把交互先打磨顺。

## 9. 启动方式建议
- 日常使用：双击 `start_gui.vbs`
- 兼容入口：双击 `start_gui.bat`
- 之后若要更像正式程序，可以做桌面快捷方式或打包成 `.exe`
