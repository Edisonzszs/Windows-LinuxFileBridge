# Windows-Linux 文件桥设计规划

## 1. 当前定位

这是一个面向个人高频办公与开发场景的 Windows 本地文件桥工具。目标不是“功能尽可能多”，而是“Windows 和 WSL Linux 之间最常见的文件操作足够顺手、足够安全、足够清晰”。

## 2. 设计目标

- 少操作：高频动作尽量 1 到 2 次点击完成
- 少判断：界面让用户一眼知道“源在哪、目标在哪、下一步做什么”
- 更安全：删除、批量操作、失败重试必须有明确反馈
- 更像桌面工具：布局稳定、状态明确、启动干净

## 3. 当前信息架构

### 3.1 主界面分区

- 顶部 Hero 区
  - 当前 WSL 发行版
  - 刷新入口
  - 状态徽标
  - 当前发行版 / Windows 面板 / Linux 面板 / 最近结果状态卡
- 中间工作区
  - 左侧 Windows 工作区
  - 中间传输中心
  - 右侧 Linux 工作区
- 底部辅助区
  - 拖拽投递区域
- 二级界面
  - “操作记录”窗口，负责展示历史记录、失败重试、清空历史

### 3.2 交互原则

- 双击文件夹进入
- 单击选中
- 右键执行管理动作
- 主按钮只做传输，不混合管理操作
- 历史记录从主界面移出，减少主视图拥挤
- 危险动作统一弹窗确认

## 5. 当前版本已解决的问题

- 传输、浏览、管理动作不再分散在不同区域
- 主界面对“当前发行版、当前面板状态、最近结果”有更清晰反馈
- 文件列表从简单列表升级为更易读的列式信息
- 右键菜单已经承担主要管理操作入口
- 历史记录不再挤占主界面，而是放到二级窗口中展示
- 新建文件夹、失败重试等操作形成了更完整的闭环

## 6. 仍值得继续优化的方向

- 更细腻的拖拽反馈和高亮提示
- 收藏目录 / 最近目录
- 批量选择辅助操作
- 更明确的传输进度展示
- 更精细的错误详情面板

## 7. 技术演进建议

### 继续用 Tkinter 的优点

- 已有代码可继续复用
- 适合个人桌面工具
- 依赖少，启动快
- 打包链路简单

### 如果追求更现代体验

后续可以迁移到以下方案之一：

- PySide6：更像正式桌面软件，控件更丰富
- Tauri + Web 前端：界面最灵活，但开发复杂度更高

当前阶段继续使用 Tkinter 仍然是合理选择，重点应该放在交互打磨与流程闭环。

## 8. 启动方式建议

- 日常使用：双击 `start_gui.vbs`
- 开发调试：运行 `python wsl_file_bridge_gui.py`
- 对外分发：优先提供单文件 EXE
